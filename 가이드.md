# 인생네컷 앱 — 올인원 가이드

이 문서 하나로 처음부터 끝까지 다 되게 만들었어요. **위에서부터 순서대로 읽으면 됩니다.**
필터·이모지·프레임·폰트 추가는 대부분 AI 없이 메모장만으로 가능해요. AI(챗지피티)는 정말 필요할 때만 짧게 요청해서 토큰을 아끼세요.

# 파트 1. GitHub 공유받고 들어가는 법

## 1-1. 초대 수락하기

1. 초대 메일(또는 문자/카톡으로 받은 초대 링크) 확인 → **"View invitation"** 또는 **"Accept invitation"** 클릭
2. GitHub 계정이 없으면 그 자리에서 새로 가입 (이메일만 있으면 5분), 있으면 로그인
3. 수락하면 저장소가 내 GitHub 화면에도 보이게 됨

## 1-2. 저장소(코드) 보는 법

- 주소창에 `github.com/songolder/life4cut` 입력해서 바로 들어갈 수 있어요
- 여기 보이는 `index.html` 등은 **코드**예요. 일반 사람들이 쓰는 앱 화면이 아니라, "이 앱을 만드는 설계도"라고 생각하면 돼요.

## 1-3. 실제로 사람들이 쓰는 앱 링크 (헷갈리지 않게 구분!)

GitHub 저장소(코드)와 실제 앱은 **다른 주소**예요.

- **코드 보는 곳** (개발자만): `github.com/songolder/life4cut`
- **실제 촬영 앱** (모든 사람이 쓰는 곳): `https://songolder.github.io/life4cut/`

행사에서 손님들에게 공유할 땐 **무조건 아래(`.github.io`) 링크만** 보내주면 돼요. 위 GitHub 저장소 링크를 보내면 안 됩니다 (코드가 보이는 개발자 페이지라서 혼란스러워요).

# 파트 2. 아이폰 vs 안드로이드 — 기기별 주의사항

이 앱은 두 기기 모두에서 작동하도록 만들었지만, 브라우저 특성상 몇 가지 다르게 동작하는 부분이 있어요. (아이폰에서 직접 테스트해본 게 아니라 일반적으로 알려진 차이를 기준으로 정리했어요. 실제로 문제가 생기면 아래 대처법부터 시도해보세요.)

## 공통 (둘 다 똑같이 잘 됨)
- 카메라 켜기, 사진 촬영, 8장 중 4장 고르기, 프레임/스티커/글자/손글씨 꾸미기, PNG 이미지 다운로드는 **아이폰·안드로이드 상관없이 잘 작동**해요.

## 브라우저 추천
- **안드로이드**: 크롬(Chrome) 사용 권장
- **아이폰**: **사파리(Safari) 사용 권장.** (아이폰에서는 크롬을 깔아도 내부적으로 애플의 엔진을 쓰기 때문에 사파리랑 큰 차이는 없지만, 그래도 가장 많이 테스트되는 조합은 사파리예요.)

## 차이가 날 수 있는 부분

| 기능 | 안드로이드(크롬) | 아이폰(사파리) |
|---|---|---|
| 촬영 과정 영상 녹화 | 잘 됨 | 최신 아이폰(iOS 14.3 이상)은 됨, 아주 오래된 기종은 이 기능만 조용히 안 뜰 수 있음 (에러 없이 그냥 생략됨) |
| 영상 파일 형식 | webm | mp4 (자동으로 맞춰서 저장됨) |
| 영상 다운로드 | 새 탭 열림 → 다운로드 아이콘(⬇) 누르기 | 새 탭에서 영상이 뜨면, 화면을 **길게 눌러서 "비디오 저장"** 선택 |
| 사진(PNG) 다운로드 | 대부분 자동으로 다운로드 폴더에 저장 | 저장 팝업이 안 뜨면, 완성된 사진을 **길게 눌러서 "사진에 저장"** 선택 |
| 필터 효과 | 잘 됨 | 최신 기종은 잘 됨, 아주 오래된 기종은 필터만 안 먹힐 수 있음 (사진 촬영 자체는 정상) |

## 문제 생겼을 때 공통 대처법
1. **길게 누르기(롱프레스)가 만능 해결책이에요.** 다운로드 버튼이 안 먹히면, 결과물(사진/영상) 자체를 손가락으로 길게 눌러서 "저장" 메뉴를 찾아보세요.
2. 카메라가 아예 안 켜지면 → 브라우저 설정에서 카메라 권한을 허용했는지 확인
3. 그래도 안 되면 → 화면을 캡처해서 챗지피티나 저(Claude)에게 보여주면 원인 파악이 훨씬 빨라요

# 파트 3. 코드 수정하는 법 (개발 이어받는 사람용)

## 3-1. 파일 여는 법 (GitHub에서 직접 수정)

1. GitHub 저장소에서 `index.html` 파일 클릭
2. 오른쪽 위 연필 아이콘(✏️) 클릭 → 편집 모드로 열림
3. 아래 "파트 4"에서 설명하는 부분만 찾아서 고치기
4. 다 고쳤으면 아래로 스크롤 → **Commit changes** 클릭

코드 맨 위쪽, `const APP_CONFIG = {` 부분부터 대부분의 수정이 이루어져요.

## 3-2. 다운받은 html 파일 내용을 컴퓨터에서 보는 법

`life4cut.html` 파일을 더블클릭하면 자동으로 **브라우저(크롬)로 실행**돼요 (코드가 아니라 앱 화면이 뜸). 코드 내용을 보려면 아래 방법 중 하나로 여세요.

**방법 A. 메모장으로 끌어다 놓기 (제일 쉬움)**
1. 시작 메뉴에서 "메모장" 실행 (빈 화면 하나 열어두기)
2. `life4cut.html` 파일을 마우스로 끌어서 **메모장 창 위에 놓기** (드래그 앤 드롭)
3. 코드 내용이 텍스트로 보이면 전체 선택(Ctrl+A) → 복사(Ctrl+C)

**방법 B. 마우스 오른쪽 클릭으로 열기**
1. 파일에서 마우스 오른쪽 클릭 → **"연결 프로그램"** 클릭
2. 목록에서 **"메모장"** 선택 (없으면 "다른 앱 선택"에서 찾기)
3. ⚠️ **"항상 이 앱을 사용하여 .html 파일 열기"에는 체크하지 마세요.** 체크하면 앞으로 html 파일을 더블클릭할 때마다 계속 메모장으로만 열려서, 정작 앱을 실행해서 테스트해보고 싶을 때 불편해져요.

## 3-3. 챗지피티에게 처음 부탁할 때 쓰는 기본 프롬프트

새로 챗지피티 채팅을 열고 수정을 부탁할 때, **바로 요청하지 말고 아래 프롬프트를 먼저 붙여넣은 뒤** 그 아래에 현재 코드를 붙여넣으세요. 챗지피티가 엉뚱한 방향(여러 파일로 쪼개기, PWA 제안 등)으로 안 새게 잡아주는 역할이에요.

```
너는 지금부터 이 HTML 파일 하나만 계속 수정하는 역할이야. 아래 규칙을 반드시 지켜줘.

[규칙]
1. 이 프로젝트는 파일 하나(html 파일 안에 CSS, JS 다 포함)로만 되어 있어.
   절대 여러 파일(index.html, style.css, app.js, config.js, manifest.json 등)로 나누지 마.
   나는 코딩을 몰라서 파일 하나만 다룰 수 있어.

2. 내가 기능 추가나 수정을 요청하면, 설명만 하지 말고
   반드시 "수정된 전체 HTML 코드"를 처음부터 끝까지 다 보여줘.
   일부분만 보여주면 내가 합칠 수가 없어.

3. 기존에 잘 작동하던 기능(카메라, 촬영, 프레임, 스티커, 손글씨, 필터, 영상 녹화,
   다운로드 등)은 건드리지 말고 그대로 유지한 채로, 요청한 부분만 추가/수정해줘.

4. PWA, 앱 설치, 폴더 자동 인식, service-worker 같은 복잡한 기능은 절대 제안하지 마.
   이건 그냥 웹페이지 하나로 충분해.

5. 코드를 준 다음에는, 내가 이 코드를 GitHub Pages에 올려서 배포할 거라는 걸 기억해줘.
   파일은 항상 index.html이라는 이름으로 유지해야 해. 다른 배포 방식은 제안하지 마.

6. 한 번에 하나의 기능만 요청할 거야. 내가 다음 요청을 하기 전까지 기다려줘.

이제 아래에 현재 코드를 붙여넣을게. 이 코드를 기준으로 이어서 작업해줘.

[여기에 현재 index.html 전체 내용 붙여넣기]
```

이 프롬프트는 짧아서 토큰을 많이 안 써요. 오히려 이걸 안 쓰고 바로 요청했다가 챗지피티가 코드를 쪼개버리면, 되돌리느라 훨씬 많은 토큰을 쓰게 되니 꼭 먼저 붙여넣으세요.

# 파트 4. 기능별 추가 가이드

## 4-1. 필터 추가하기 (AI 필요 없음)

`APP_CONFIG.filters` 배열을 찾으세요.

```js
filters: [
  { id:'none', name:'필터 없음', css:'none' },
  { id:'bw', name:'흑백', css:'grayscale(1)' },
  { id:'sepia', name:'세피아', css:'sepia(.7)' },
  { id:'bright', name:'화사하게', css:'brightness(1.15) contrast(1.05)' },
  { id:'vivid', name:'비비드', css:'contrast(1.2) saturate(1.3)' },
  { id:'soft', name:'뽀샤샤', css:'brightness(1.12) contrast(.9) saturate(1.05) blur(0.6px)' }
],
```

중괄호 `{ }` 하나를 통째로 복사해서 붙여넣고 `id`, `name`, `css` 값만 바꾸세요.

자주 쓰는 값들:
| 효과 | css 값 |
|---|---|
| 밝게 | `brightness(1.2)` |
| 어둡게 | `brightness(0.85)` |
| 색 진하게 | `saturate(1.4)` |
| 색 빼기 | `saturate(0.5)` |
| 대비 세게 | `contrast(1.3)` |
| 살짝 흐리게 | `blur(1px)` |
| 색조 바꾸기 | `hue-rotate(30deg)` |

여러 개를 띄어쓰기로 이어붙이면 (예: `brightness(1.1) sepia(.3)`) 한 번에 합쳐진 효과가 나와요.

## 4-2. 이모지 스티커 추가하기 (AI 필요 없음)

`APP_CONFIG.stickers` 배열을 찾으세요.

```js
stickers: ['❤️','💕','✨','⭐','🎀','🌸','🌈','🐰','🐻','😆','✌️','🫶','🎉','☁️','🍀','💙','💜','🖤','✝️','⛺','📖','🌟'],
```

원하는 이모지를 따옴표(`' '`) 안에 넣어서 쉼표로 추가하면 끝이에요.

**이모지 고르는 법**: 윈도우 `Win + .` / 아이폰·맥 `control + cmd + space` / 또는 [emojipedia.org](https://emojipedia.org)에서 복사

## 4-3. 프레임(디자인) 추가하기 (AI 필요 없음)

`APP_CONFIG.frames` 배열에서 객체 하나를 통째로 복사해서 값만 바꾸면 돼요.

```js
{ id:'my-frame', name:'새 프레임 이름', bg:'#색상코드', accent:'#포인트색상', title:'기본 문구', deco:['🎈','🎈'] }
```

- `bg`: 배경색 / `accent`: 테두리·글자색 / `title`: 기본 문구 / `deco`: 모서리 이모지 2개 (없으면 `['','']`)

색상 코드는 [htmlcolorcodes.com](https://htmlcolorcodes.com)에서 골라서 복사하면 돼요.

## 4-4. 진짜 디자인된 PNG 프레임 넣기 (미리캔버스/캔바 활용)

**중요**: 우리 앱은 레이아웃 3종(세로 스트립/2×2 격자/가로 스트립) × 사진 모양 2종(가로형/세로형) = **총 6가지 배치 조합**이 있어요. PNG 프레임 하나는 딱 한 가지 조합에만 맞기 때문에, **여러 조합에서 다 예쁘게 보이게 하려면 조합별로 PNG를 각각 만들어야 해요.**

물론 전부 다 만들 필요는 없어요. 예를 들어 "세로 스트립 + 가로형 사진" 하나만 만들어도 되고, 나중에 필요할 때 다른 조합을 하나씩 추가해도 돼요. **안 만든 조합은 자동으로 기본 색상 테두리로 대체되니 에러 없이 자연스럽게 넘어가요.**

### 만드는 순서 — 처음부터 끝까지 하나씩 (예시: strip-v_landscape 기준)

여기서는 가장 기본 조합인 **`strip-v_landscape`** (세로 스트립 + 가로형 사진, 748×2344px) 하나를 예로 들어서, 미리캔버스 기준으로 정말 하나하나 설명할게요. 다른 조합을 만들 때도 크기와 좌표 숫자만 바꿔서 똑같은 순서로 하면 됩니다.

#### 1단계. 정확한 크기로 새 캔버스 만들기

1. 미리캔버스 접속 → **"바로 시작하기"** 또는 새 디자인 만들기
2. 오른쪽 위 또는 상단에 **"사용자 지정 크기"** (또는 "직접 입력") 옵션 찾기
3. 단위를 **px(픽셀)**로 맞추고, **가로 748 / 세로 2344** 입력해서 캔버스 생성
   (이 숫자는 위 표의 `strip-v_landscape` 캔버스 크기예요. 다른 조합을 만들 땐 그 조합의 숫자로 바꾸면 됩니다.)

#### 2단계. "사진이 보일 자리"를 안내선(가이드)으로 표시하기

사진 4장이 들어갈 자리를 미리 표시해두고 작업하면 훨씬 편해요.

1. 도형 메뉴에서 **네모(사각형)** 도형을 캔버스에 하나 추가
2. 그 사각형을 선택한 상태에서, 화면에 보이는 **위치(X, Y)** 와 **크기(W, H)** 입력창을 찾기
   (미리캔버스는 요소를 클릭하면 상단이나 오른쪽에 정확한 X/Y/크기 숫자를 직접 입력할 수 있는 칸이 나와요)
3. 아래 표 그대로 숫자를 입력:

   | 몇 번째 사진칸 | X | Y | 가로(W) | 세로(H) |
   |---|---|---|---|---|
   | 1번 | 24 | 24 | 700 | 525 |
   | 2번 | 24 | 561 | 700 | 525 |
   | 3번 | 24 | 1098 | 700 | 525 |
   | 4번 | 24 | 1635 | 700 | 525 |

4. 이 사각형 4개를 **"여기에 사진이 보여요"라고 표시해두는 안내선**으로만 쓸 거예요. 색은 아무거나 상관없어요 (예: 회색). 이 사각형 자체가 최종 결과물에 남는 게 아니라, 작업할 때 위치 참고용이에요.

#### 3단계. 안내선 "주변"에 원하는 디자인 꾸미기

1. 이제 그 사각형 4개는 그대로 두고, **그 바깥쪽 여백 공간**에 원하는 테두리·리본·스티커·패턴 등을 자유롭게 디자인하세요
2. 안내선 사각형 **안쪽 영역(사진이 보여야 하는 자리)에는 디자인 요소를 겹치지 않게** 하세요. 이 안쪽은 나중에 완전히 비워둬야(투명하게) 해요.
3. 캔버스 맨 아래쪽 160px 구간(이 조합 기준 y: 2184~2344)에는 문구·날짜 글자가 자동으로 들어가니, 이 부분은 너무 빽빽하게 채우지 마세요.

#### 4단계. 안내선 사각형 지우기, 투명 배경으로 저장

1. 디자인이 다 끝나면, 2단계에서 만들었던 **회색 안내선 사각형 4개를 전부 선택해서 삭제**하세요. (진짜로 사각형이 남아있으면 안 돼요! 그 자리가 이제 "빈 공간"이 되어야 해요.)
2. 사각형을 지운 그 자리는 미리캔버스에서 그냥 **캔버스의 원래 배경**으로 보일 거예요. 다운로드할 때 배경을 투명하게 설정하면, 이 빈 공간이 자동으로 투명하게 저장돼요.
3. 오른쪽 위(또는 상단) **"다운로드"** 버튼 클릭
4. 파일 형식에서 **"PNG"** 선택
5. **"배경 투명하게"** 옵션이 있으면 반드시 체크 (미리캔버스는 유료 기능일 수 있어요 — 없으면 캔바의 무료 "배경 제거" 기능이나, 처음부터 배경색을 아예 안 채우고 시작하는 방법도 있어요)
6. 다운로드된 파일 이름을 알아보기 쉽게 바꾸기 (예: `church-strip-v-landscape.png`)

⚠️ **투명 배경이 핵심이에요.** 안내선을 지운 자리가 투명하게 저장되지 않으면, 나중에 사진이 안 보이고 캔바 배경색만 보이게 돼요. 다운로드 후 파일을 열어서 사진 자리가 정말 비어있는지(체크무늬 배경으로 보이는지) 확인하세요.

#### 5단계. GitHub에 업로드

1. GitHub 저장소 페이지에서 **"Add file" → "Upload files"**
2. 방금 만든 PNG 파일을 드래그해서 업로드 (파일명은 4단계에서 정한 그대로, `index.html`과 **같은 위치**에)
3. Commit changes

#### 6단계. 코드에서 프레임과 연결하기

1. `index.html` 파일 열기 → 연필 아이콘(✏️) 클릭
2. `APP_CONFIG.frames` 배열 찾기 (코드 위쪽에 있어요)
3. 아래처럼 새 프레임 하나를 배열 안에 추가:

   ```js
   {
     id:'church-frame',
     name:'교회 프레임',
     bg:'#ffffff',
     accent:'#333333',
     title:'여름성경학교',
     deco:['',''],
     overlays: {
       'strip-v_landscape': 'church-strip-v-landscape.png'
     }
   }
   ```

   각 부분이 뭔지 하나씩 설명하면:
   - `id`: 이 프레임을 구분하는 이름표 (다른 프레임과 겹치지만 않으면 아무 영어 단어나 가능)
   - `name`: 프레임 선택 화면에 보일 이름
   - `bg`, `accent`: PNG가 안 뜰 때(파일명이 틀렸거나 아직 안 올렸을 때) 대신 보여줄 기본 색상 — 안전장치라고 생각하면 돼요
   - `title`: 이 프레임을 골랐을 때 기본으로 들어갈 문구
   - `deco`: PNG를 쓰는 프레임은 보통 빈 칸(`['','']`)으로 둬요 (PNG 안에 이미 장식이 있으니까)
   - `overlays`: **여기가 핵심.** `'strip-v_landscape'` 라는 이름(=지금 만든 조합의 이름)에, 방금 업로드한 파일명을 정확히 그대로 적어주면 돼요. **파일명 철자가 하나라도 다르면 안 뜨니, 업로드한 실제 파일명을 복사해서 붙여넣는 게 제일 안전해요.**

4. 이미 있던 프레임들 목록(`classic-white`, `signature-pink` 등) 사이에 쉼표(`,`)로 구분해서 이 블록을 끼워넣기
5. 아래로 스크롤 → **Commit changes**

#### 7단계. 확인

앱 링크로 들어가서 설정 화면 → 프레임 선택 목록에 "교회 프레임"이 새로 보이고, 미리보기에 **"PNG"** 배지가 붙어있는지 확인하세요. 선택하고 촬영까지 해보면 실제로 사진 위에 PNG 디자인이 정확히 겹쳐서 나오는지 확인할 수 있어요.

#### 다른 조합도 추가하고 싶다면

위 1~6단계를 그대로 반복하되, 표에서 원하는 다른 조합(예: `grid_landscape`)의 크기·좌표 숫자로 바꿔서 새 PNG를 하나 더 만들고, 6단계에서 `overlays` 안에 **줄 하나만 추가**하면 돼요:

```js
overlays: {
  'strip-v_landscape': 'church-strip-v-landscape.png',
  'grid_landscape': 'church-grid-landscape.png'
}
```

## 4-5. 이미지 스티커 영구 등록하기 (AI 필요 없음)

1. 배경이 투명한 PNG(또는 WebP) 파일 준비
2. `index.html`과 같은 폴더에 업로드
3. `APP_CONFIG.imageStickers` 배열에 한 줄 추가:
   ```js
   imageStickers: [
     { id:'church-logo', name:'교회 로고', file:'church-logo.png' },
   ],
   ```

파일명만 정확히 맞으면 자동으로 스티커 목록에 나타나요. 파일이 없거나 이름이 틀려도 에러 없이 목록에 안 보일 뿐이니 안심하세요.

**참고**: 앱 안 "스티커" 탭의 "내 사진/로고를 스티커로 추가" 버튼은 그 자리에서 임시로만 쓰는 용도예요 (새로고침하면 사라짐). 위 방법이 영구 등록이에요.

## 4-6. 폰트(글꼴) 넣기 — 두 가지 방법

**방법 A. 지금 바로 미리보기만 (AI 필요 없음)**
앱 "설정" 화면의 **"폰트 파일 불러오기"** 버튼으로 폰트 파일(.woff2, .ttf, .otf) 선택 → 즉시 적용되지만 **내 화면에서만** 보여요. 새로고침하면 원래대로 돌아가요.

**방법 B. 모든 사람에게 영구 적용**
1. 눈누(noonnu.cc)에서 폰트 파일(.woff2 추천) 다운로드
2. `index.html`과 같은 위치에 함께 업로드
3. 코드 맨 위쪽 `@font-face` 부분에서 `url('./CustomFont.woff2')`를 실제 업로드한 파일 이름과 똑같이 맞추기

# 파트 5. 무료 AI(챗지피티) 토큰 아끼는 팁

1. **파트 4의 모든 항목은 AI 없이 이 문서만 보고 직접 해보세요.** 대부분 배열에 값 하나 추가하거나 파일명만 맞추는 것뿐이라 실수해도 되돌리기 쉬워요.
2. 정말 AI가 필요할 때는:
   - **"이 부분만 바꿔줘"** 식으로 구체적으로 요청하기 (전체 설명 반복 X)
   - 코드 전체를 붙여넣고 "이 코드를 기준으로 OO 부분만 수정한 전체 코드를 줘"라고 요청하면 한 번에 끝남
   - 한 번에 여러 기능을 몰아서 요청하지 말고, **하나씩 요청하고 테스트한 뒤 다음 요청**하기
3. 대화가 너무 길어져서 AI가 이상해지면, **새 대화를 열고 코드 전체를 다시 붙여넣은 뒤** 이어서 요청하세요.
